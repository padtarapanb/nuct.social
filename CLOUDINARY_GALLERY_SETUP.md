# ตั้งค่า Gallery ให้ดึงรูปจาก Cloudinary อัตโนมัติ

ฟีเจอร์นี้ทำให้หน้า Gallery ดึงรูป**ทั้งโฟลเดอร์**จาก Cloudinary มาแสดงเองอัตโนมัติ
อัปโหลดรูปใหม่เข้าโฟลเดอร์ที่กำหนด → เว็บแสดงรูปใหม่ทันที ไม่ต้องเข้า `/admin` ไม่ต้องแก้โค้ด ไม่ต้อง deploy ใหม่

## ไฟล์ที่เพิ่ม/แก้ไข

1. **เพิ่มใหม่** `api/gallery.js` — Vercel Serverless Function เรียก Cloudinary Admin API
2. **เพิ่มใหม่** `src/hooks/useCloudinaryGallery.js` — hook ฝั่ง React เรียก `/api/gallery`
3. **แก้ไข** `src/components/landing/Gallery.jsx` — ใช้รูปจาก Cloudinary แทน (ถ้ามี) โดยยัง fallback ไปใช้ข้อมูลจาก Supabase/admin เหมือนเดิมถ้ายังไม่ได้ตั้งค่า Cloudinary หรือโฟลเดอร์ว่าง

## ขั้นตอนที่ 1: หาค่า Cloudinary 3 ตัว

1. ล็อกอิน https://cloudinary.com/console
2. หน้า Dashboard จะเห็น **Cloud Name**, **API Key**, **API Secret** (กดปุ่มตาเพื่อดู Secret)
3. สร้างโฟลเดอร์สำหรับเก็บรูป Gallery เช่น ตั้งชื่อว่า `nuct-gallery` (Media Library > Create folder)

## ขั้นตอนที่ 2: ตั้งค่า Environment Variables บน Vercel

ไปที่โปรเจกต์บน Vercel > **Settings** > **Environment Variables** แล้วเพิ่ม:

| Key | Value |
|---|---|
| `CLOUDINARY_CLOUD_NAME` | ชื่อ cloud ของคุณ |
| `CLOUDINARY_API_KEY` | API Key |
| `CLOUDINARY_API_SECRET` | API Secret |
| `CLOUDINARY_FOLDER` | ชื่อโฟลเดอร์ เช่น `nuct-gallery` |

ใส่ครบแล้วกด **Redeploy** โปรเจกต์ 1 ครั้ง (env var ใหม่จะมีผลตอน build/run ครั้งถัดไปเท่านั้น)

> ⚠️ ห้ามใส่ค่าพวกนี้ในไฟล์ที่ขึ้นต้นด้วย `VITE_` หรือไฟล์ฝั่ง frontend เด็ดขาด เพราะ `API_SECRET` ต้องอยู่ฝั่ง server (`api/gallery.js`) เท่านั้น ไม่งั้นจะรั่วไปอยู่ในโค้ดที่ผู้ใช้ทุกคนมองเห็นได้

## ขั้นตอนที่ 3: ทดสอบ

- **บนเว็บจริง (Vercel):** เข้าเว็บ แล้วลองอัปโหลดรูปใหม่เข้าโฟลเดอร์ที่ตั้งไว้ใน Cloudinary รีเฟรชหน้าเว็บ ควรเห็นรูปใหม่ในหมวด Gallery ทันที
- **ในเครื่องตัวเอง:** `npm run dev` (Vite) เพียงอย่างเดียว**จะไม่รัน** `/api/gallery` ให้ (Vite ไม่รู้จัก Vercel Functions) ต้องใช้คำสั่งนี้แทนเพื่อทดสอบ API ในเครื่อง:
  ```bash
  npm install -g vercel
  vercel dev
  ```
  แล้วต้องสร้างไฟล์ `.env` ที่มี 4 ค่าด้านบนไว้ในเครื่องด้วย (หรือรัน `vercel env pull` ถ้า link โปรเจกต์กับ Vercel ไว้แล้ว)

## หมายเหตุ

- ถ้ายังไม่ตั้งค่า Cloudinary เว็บจะยังทำงานปกติ — หน้า Gallery จะกลับไปใช้รูปที่แอดมินตั้งไว้ผ่าน `/admin` (Supabase) เหมือนเดิม ไม่พังแน่นอน
- ชื่อรูปที่แสดง (title ใต้รูป) จะถูกตั้งจาก**ชื่อไฟล์**ใน Cloudinary อัตโนมัติ (ตัด `-`/`_` เป็นช่องว่าง) ถ้าอยากตั้งชื่อรูปเอง แนะนำตั้งชื่อไฟล์ให้สื่อความหมายก่อนอัปโหลด เช่น `ค่ายฤดูร้อน-2025.jpg` → จะโชว์เป็น "ค่ายฤดูร้อน 2025"
  - ถ้าชื่อไฟล์เป็นแบบที่กล้อง/มือถือตั้งให้อัตโนมัติ (เช่น `IMG_20250612_143210` หรือตัวเลขล้วน ๆ) ระบบจะไม่โชว์ชื่อใต้รูป (ปล่อยว่างไว้) แทนที่จะโชว์ตัวเลขที่ไม่มีความหมาย
- ผลลัพธ์จาก API จะถูกแคชไว้ 30 วินาทีเพื่อลดการเรียก Cloudinary ถี่เกินไป ถ้าอัปรูปแล้วยังไม่เห็น ลองรอสักครู่แล้วรีเฟรชอีกที

## กรณีต่าง ๆ ตอนแอดมิน "เพิ่มรูป" ที่ระบบป้องกันไว้ให้แล้ว

- **อัปโหลดไฟล์ที่ไม่ใช่รูป** (วิดีโอ, PDF, ไฟล์เอกสาร) เข้าโฟลเดอร์เดียวกันโดยไม่ตั้งใจ → ระบบกรองเอาเฉพาะรูปภาพจริงเท่านั้น ไฟล์อื่นจะไม่โผล่มาในหน้า Gallery
- **รูปเยอะมาก (เกิน 200 รูป)** → ระบบจะดึงมาให้ครบทุกหน้า (pagination) จนถึงเพดานสูงสุด 1,000 รูป ถ้าคาดว่าจะมีรูปเกิน 1,000 รูปในโฟลเดอร์เดียว แจ้งได้ ผมปรับเพดานให้ได้
- **ไฟล์รูปขนาดใหญ่ (ถ่ายจากมือถือ เช่น 8-10MB/รูป)** → หน้ากริดจะขอ "รูปย่อ" จาก Cloudinary (ขนาด 500x500, บีบอัดอัตโนมัติ) แทนรูปเต็ม ทำให้หน้าเว็บโหลดเร็ว ส่วนตอนกดดูรูปขยาย (lightbox) จะใช้รูปเต็มตามปกติ
- **นามสกุลไฟล์แปลก ๆ** (เช่น .psd, .ai ที่เปิดในเบราว์เซอร์ไม่ได้) → ถูกกรองออก ไม่ให้ขึ้นเป็นรูปแตก
- **ชื่อไฟล์อัตโนมัติจากกล้อง/มือถือ** → ไม่โชว์ป้ายชื่อที่ไม่มีความหมายใต้รูป (ดูหัวข้อด้านบน)
