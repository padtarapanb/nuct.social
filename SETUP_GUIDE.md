# คู่มือติดตั้งระบบหลังบ้าน (แอดมิน) — ทำตามทีละขั้นตอน

เว็บนี้ตอนนี้มี **หน้าแอดมิน** ให้เพิ่ม/ลบ/แก้ไขข้อมูลได้เองที่ `/admin` โดยไม่ต้องแตะโค้ดเลย
แต่ต้องตั้งค่า "ที่เก็บข้อมูล" ก่อน 1 ครั้ง (ใช้ Supabase ซึ่งฟรี) ใช้เวลาประมาณ 15-20 นาที

ถ้ายังไม่ตั้งค่าตามคู่มือนี้ เว็บจะยังใช้งานได้ปกติ (โชว์ข้อมูลตัวอย่างเดิม) แต่หน้า `/admin` จะยังบันทึกอะไรไม่ได้

---

## ขั้นตอนที่ 1: สร้างบัญชี Supabase (ฟรี)

1. ไปที่ https://supabase.com แล้วกด **Start your project**
2. สมัคร/ล็อกอินด้วย GitHub หรืออีเมลก็ได้
3. กด **New Project**
   - ตั้งชื่อโปรเจกต์ เช่น `thepr-nuct`
   - ตั้ง Database Password (จดเก็บไว้ ไม่ต้องใช้บ่อยแต่เผื่อไว้)
   - เลือก Region ใกล้ไทยที่สุด เช่น Singapore
   - กด **Create new project** แล้วรอสัก 1-2 นาทีให้ระบบสร้างเสร็จ

## ขั้นตอนที่ 2: สร้างตารางข้อมูล (รันไฟล์ SQL)

1. ในเมนูซ้ายของ Supabase กด **SQL Editor** > **New query**
2. เปิดไฟล์ `supabase/schema.sql` ที่แนบมาในโปรเจกต์นี้ คัดลอกทั้งหมด
3. วางลงในช่อง SQL Editor แล้วกด **Run**
4. ถ้าขึ้น "Success. No rows returned" แปลว่าเรียบร้อย — ตารางทั้งหมดถูกสร้างพร้อมข้อมูลตัวอย่างแล้ว

## ขั้นตอนที่ 3: สร้างบัญชีแอดมินสำหรับล็อกอิน

1. ในเมนูซ้าย กด **Authentication** > **Users**
2. กด **Add user** > **Create new user**
3. ใส่อีเมลและรหัสผ่านของแอดมิน (จำไว้ ใช้ล็อกอินหน้า `/admin`)
4. ปิดตัวเลือก "Auto Confirm User" ให้เป็น **เปิด** (ติ๊กถูก) เพื่อให้ใช้ได้ทันทีโดยไม่ต้องยืนยันอีเมล
5. กด **Create user**

> อยากเพิ่มแอดมินหลายคน ทำซ้ำขั้นตอนนี้ได้เรื่อย ๆ แต่ละคนใช้อีเมล/รหัสผ่านของตัวเอง

## ขั้นตอนที่ 4: คัดลอกกุญแจเชื่อมต่อ (API keys)

1. ในเมนูซ้าย กด **Project Settings** (รูปเฟือง) > **API**
2. จะเห็น 2 ค่าที่ต้องใช้:
   - **Project URL** (หน้าตาแบบ `https://xxxxxxxxxxxx.supabase.co`)
   - **anon public** key (สตริงยาว ๆ ใต้หัวข้อ Project API keys)
3. เก็บ 2 ค่านี้ไว้ก่อน จะใช้ในขั้นตอนถัดไป

## ขั้นตอนที่ 5: ใส่ค่าเชื่อมต่อในโปรเจกต์ (สำหรับรันดูในเครื่องตัวเอง)

1. คัดลอกไฟล์ `.env.example` แล้วเปลี่ยนชื่อเป็น `.env`
2. แก้ค่าในไฟล์ให้เป็นค่าจริงจากขั้นตอนที่ 4:
   ```
   VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=ค่า-anon-public-key-ของจริง
   ```
3. รันดูผลในเครื่อง:
   ```bash
   npm install
   npm run dev
   ```
4. เปิด `http://localhost:5173/admin` ลองล็อกอินด้วยอีเมล/รหัสผ่านที่สร้างไว้ในขั้นตอนที่ 3

## ขั้นตอนที่ 6: ตั้งค่าบน Vercel (เว็บจริงที่ deploy อยู่)

เนื่องจากมีบัญชี Vercel ที่ deploy เว็บนี้อยู่แล้ว ให้ทำดังนี้:

1. เข้า https://vercel.com/dashboard แล้วเลือกโปรเจกต์เว็บนี้ (the-pr-nuct)
2. ไปที่แท็บ **Settings** > **Environment Variables**
3. เพิ่มตัวแปร 2 ตัว (เลือก Environment เป็น Production, Preview, Development ทั้งหมด):
   | Key | Value |
   |---|---|
   | `VITE_SUPABASE_URL` | Project URL จากขั้นตอนที่ 4 |
   | `VITE_SUPABASE_ANON_KEY` | anon public key จากขั้นตอนที่ 4 |
4. กด **Save**
5. อัปโหลดโค้ดชุดใหม่นี้ขึ้น GitHub repo เดิม (หรือ upload ตรง ๆ ผ่าน Vercel ก็ได้) แล้วไปที่แท็บ **Deployments** กด **Redeploy** ที่ deployment ล่าสุด (ต้อง Redeploy ใหม่ทุกครั้งหลังเพิ่ม/แก้ env variable)
6. รอ build เสร็จ แล้วเข้า `https://the-pr-nuct.vercel.app/admin` เพื่อล็อกอินแอดมินได้เลย

## วิธีใช้งานหน้าแอดมิน (`/admin`)

- ล็อกอินด้วยอีเมล/รหัสผ่านที่สร้างไว้ในขั้นตอนที่ 3
- เมนูซ้ายมีหัวข้อ: ข้อความทั่วไป, กิจกรรม, อีเวนต์, Connect With Us, แกลเลอรี, ทีมงาน, เสียงจากสมาชิก, FAQ
- แต่ละหัวข้อ **เพิ่ม/แก้ไข/ลบ/จัดลำดับ** ได้เองทั้งหมด กดบันทึกแล้วเว็บจริงจะอัปเดตทันที (รีเฟรชหน้าเว็บ)
- ช่องที่ต้องการใส่ "รูปภาพ" (แกลเลอรี, รูปทีมงาน) ให้ใส่เป็น **ลิงก์ URL รูปภาพ** ที่ขึ้นต้นด้วย `http://` หรือ `https://` เช่น ลิงก์จาก Google Drive ที่แชร์แบบสาธารณะ, Imgur, หรือ Facebook — ถ้าไม่ใส่จะขึ้นเป็นการ์ดสีไล่เฉด/ตัวอักษรย่อแทน

## เกี่ยวกับข้อมูลจากเว็บเก่า (Google Sites)

ลองดึงข้อมูลจากเว็บเก่า (https://sites.google.com/view/nuchristianclub) ให้แล้ว แต่เว็บ Google Sites ใช้ JavaScript render เนื้อหา ทำให้ระบบดึงข้อความจริง (ชื่อทีมงาน, คำอธิบายกิจกรรม, รูปภาพ ฯลฯ) ออกมาไม่ได้โดยอัตโนมัติ

**แนะนำ:** เปิดเว็บเก่าเทียบไปด้วย แล้ว copy ข้อความ/ลิงก์รูปที่ต้องการ มาใส่ในหน้าแอดมินของเว็บใหม่ทีละหัวข้อ (ใช้เวลาไม่นานเพราะเพิ่ม/แก้ได้จากหน้าเว็บโดยตรงแล้ว)
